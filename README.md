# CS-340-Project Two: Grazioso Salvare 
I write readable and maintainable code by structuring it cleanly and dividing up what is responsible for each piece of code. The first project provided me with a CRUD (create-read-update-delete) python module that handled all database interactions. The second project allowed me to reuse the same CRUD python module and connect it to the MongoDB database for the dashboard. This helped keep the design of the dashboard separate from the logic that connects to the database, making both easier to modify when needed. It will also make the program able to be modified as needed, since the CRUD module can be used again in other future projects that require database interaction.

When I am working through a problem, I usually begin by checking the requirements, breaking them down into smaller steps, and testing each one at a time. With the project I started by testing and designing the database queries prior to developing the functionality of the dashboard. There were many different things I had to plan for this project since it combined backend development and frontend development, which was a lot of planning compared to previous assignments. The computer science field provides companies with the tools to effectively manage data. As such, the dashboard will help Grazioso Salvare quickly identify dogs that have met the training criteria and assist in the company's decision making process.
